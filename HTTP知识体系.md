# 01-HTPP报文结构是怎么样的
> header + body的结构，具体而言：起始行+头部+空行+实体
## 起始行
对于请求报文来说，起始行类似于下面这样
```
GET /home HTTP/1.1
```
也就是方法+路径+http版本
对于响应报文来说，起始行一般长这样
```
HTTP/1.1 200 OK
```
http版本+状态码+原因
## 头部
头部字段的格式

- 1. 字段名不区分大小写
- 2. 字段名不允许出现空格，不可以出现下划线
- 3. 字段名后面必须紧接着:
## 空行
用来区分头部和实体
## 实体
就是具体的数据了，也就是body部分。请求报文对应的就是请求体，响应报文对应的就是响应体。
